자료 전달
→ 값 / 참조 / 포인터 로 전달하는 3가지 메커니즘으로 구분된다.
1) 값으로 전달
: 인수의 값이 복사되어서 매개변수에 할당된다
  호출되는 함수 쪽에서 인수를 변경하지 않게 만들고 싶을 때 사용
  읽기 전용 접근이라고도 표현함

```cpp
/**************************************************************
 * 값으로 전달 메커니즘에서 parameters의 변경이               *
 * arguments에 영향을 주지 않는다는 것을 확인하는 프로그램    *
 **************************************************************/
#include <iostream>
using namespace std;

// 함수 선언
void fun(int y);             

int main()
{
  // 변수 선언
  int x = 10;
  // x를 arguments fun 함수 호출
  fun(x);   
  // x의 값이 변했는지 확인(변하지 않음)
  cout << "main 함수 내부의 x = " << x << endl;
  return 0;
}
/**************************************************************
 * fun 함수는 x 값을 전달 받고 이를 y에 저장                  *
 * 이때 x와 y는 서로 다른 메모리 위치를 가짐                  * 
 * 따라서 fun 함수의 y를 증가시켜도                           * 
 * main 함수의 x에는 영향 없음                                * 
 **************************************************************/
void fun(int y)    
{
  y++;
  cout << "fun 함수 내부의 y = " << y << endl;
  return;
}
```

2)참조로 전달

```cpp
/**************************************************************
* 참조로 전달 메커니즘에서 parameter의 변경이                 * 
* argument에 영향을 준다는 것을 확인하는 프로그램             *
***************************************************************/
#include <iostream>
using namespace std;

// 함수 선언
void fun(int&  y); // & 기호는 y가 참조라는 것을 나타냄

int main()
{
  // 변수 선언
  int x = 10;
  // x를 argument fun 함수 호출
  fun(x); 
  // x의 값이 변했는지 확인(변함)
  cout << "main 함수 내부의 x = " << x << endl;
  return 0;
}
/**************************************************************
 * fun 함수는 y로 참조를 전달 받음                            *
 * 따라서 y는 함수 호출 때 argument의 별칭이 됨               *
 * parameter를 변경하면                                       *
 * argument가 함께 변화                                       * 
 **************************************************************/
void fun(int&  y)
{
  y++;
  cout << "fun 함수 내부의 y = " << y << endl;
  return;
}
```

함수 오버로딩

매개변수가 다르다면 같은 이름 사용가능

- 함수 시그니처
: 프로그램이 같은 이름의 함수를 허용할 때, 함수들을 구분하기 위해 사용하는 기준
함수 시그니처는 매개변수들의 자료형과 조합

스코프?
→ 특정 엔티티(상수, 변수, 객체, 함수 등)를 사용할 수 있는 범위

- 지역 스코프
→ 선언된 위치부터 블록이 끝나는 부분 내부에서 사용 가능

범위 해결 연산자 ( : : )
→ 지역 엔티티가 전역 엔티티에 접근해야 하는 경우 사용

자동 지역 변수
→ 함수가 호출될 때 생성, 종료될 때 소멸

정적 지역 변수
→ static 변경자를 앞에 붙여서 만든다.
→ 프로그램이 종료되기 전까지 유지된다.
→ 한 번만 초기화되며 프로그램이 살아있는 동안 프로그램은 메모리 위의 변수를 추적한다.
→ 함수를 여러 번 호출하면, 모든 함수들이 같은 변수를 공유한다.

- 자동 지역 변수는 초기화하지 않으면 쓰레기값을 가짐
- 전역 변수와 정적 지역 변수는 초기화 안하면 기본값을 가짐

```cpp
// circle.h
#ifndef CIRCLE_H
#define CIRCLE_H
#include <iostream>
#include <cassert>
using namespace std;
// 클래스 정의
class Circle
{
private:
    double radius;

public:
    Circle(double radius);         // 매개변수가 있는 생성자
    Circle();                      // 기본 생성자
    Circle(const Circle &circle);  // 복사 생성자
    ~Circle();                     // 소멸자
    void setRadius(double radius); // 설정자
    double getRadius() const;      // 접근자
    double getArea() const;        // 접근자
    double getPerimeter() const;   // 접근자
};
#endif
```

```cpp
// circle.cpp
#include "circle.h"
/*************************************************************
 * 매개변수가 하나 있는 생성자                               *
 * 객체를 어떤 값으로 초기화해서 생성하고 싶을 때 사용       *
 * 매개변수가 0 또는 양의 정수인지 확인하고,                 *
 * 아니라면 프로그램을 중단                                  *
 *************************************************************/
Circle::Circle(double rds)
    : radius(rds)
{
    if (radius < 0.0)
    {
        assert(false);
    }
}
/*************************************************************
 * 기본 생성자                                               *
 * 원을 반지름 0.0으로 초기화하여 생성                       *
 *************************************************************/
Circle::Circle()
    : radius(0.0)
{
}
/***********************************************************************
 * 복사 생성자이며 다른 원의 반지름을 기반으로 객체를 초기화해서 생성  *
 * 복사 대상 객체가 이미 반지름 0 또는 양의 정수라는 것이              *
 * 확인되었을 것이므로 별도의 확인을 하지 않음                         *
 ***********************************************************************/
Circle::Circle(const Circle &circle)
    : radius(circle.radius)
{
}
/*************************************************************
 * 객체가 파괴될 때                                          *
 * 객체와 관련된 것을 정리해주는 소멸자                      *
 *************************************************************/
Circle::~Circle()
{
}
/*************************************************************
 * setRadius 함수의 정의                                     *
 * 원의 반지름을 변경하고 싶을 때 사용                       *
 * 입력된 매개변수가 0 또는 양의 정수인지 확인               *
 *************************************************************/
void Circle::setRadius(double value)
{
    radius = value;
    if (radius < 0.0)
    {
        assert(false);
    }
}
/*****************************************************************
 * getRadius 함수 정의                                           *
 * 원의 반지름을 리턴                                            *
 * 호스트 객체의 데이터 멤버를 변경하지 않게 const 한정자를 붙임 *
 *****************************************************************/
double Circle::getRadius() const
{
    return radius;
}
/*****************************************************************
 * getArea 함수 정의                                             *
 * 원의 넓이를 리턴                                              *
 * 호스트 객체의 데이터 멤버를 변경하지 않게 const 한정자를 붙임 *
 *****************************************************************/
double Circle::getArea() const
{
    const double PI = 3.14;
    return (PI * radius * radius);
}
/*****************************************************************
 * getPerimeter 함수 정의                                        *
 * 원의 둘레를 리턴                                              *
 * 호스트 객체의 데이터 멤버를 변경하지 않게 const 한정자를 붙임 *
 *****************************************************************/
double Circle::getPerimeter() const
{
    const double PI = 3.14;
```

배 열

자료형
: 배열의 자료형은 요소의 자료형을 의미한다.
  배열 내부의 모든 요소는 같은 자료형을 가져야한다.

용적
: 보유할 수 있는 요소의 최대 수
  배열이 선언된 이후, 변경할 수 없는 상수이다.
  용적을 나타내는 상수는 대문자를 사용한다.

크기
: 어떤 시점에 존재하는 요소의 수(==유효한 데이터의 수)
  배열 사용 전 크기를 미리 선언해야 하며, 선언 시 컴파일러가 메모리의 특정 위치에 배열을 할당한다.
  처음 배열 선언 시 배열의 요소를 초기화할 수도 있다.

배열 선언
: 배열의 선언은 자료형, 이름, 용적, 세미콜론으로 이루어진다.
  배열의 선언은 변수 선언과 비교했을 때, 대괄호와 용적이 추가된 것을 확인할 수 있다.
  배열의 용적은 상수(또는 리터럴)이다.

초기화

1. 배열이 전역 영역에서 선언된 경우, 모든 요소는 자료형의 기본값으로 할당
bool 자료형은 false,
char 자료형은 Null 문자,
int 자료형은 0,
부동 소수점 자료형은 0.0,
객체 자료형은 기본 생성자로 생성된 객체가 기본값이다.
2. 배열이 함수 내부에서 선언된 경우 쓰레기 값으로 초기화되며,
초기화 시 명시적/암묵적/ 나머지 요소의 기본 초기화가 있다.

명시적 초기화
: 배열의 초기값을 명시적으로 지정할 수 있으며 중괄호 내부 값들을 쉼표로 구분하여 지정한다.

암묵적 초기화
: 용적을 지정하지 않고 배열을 선언할 수 있으며, 컴파일러가 알아서 용적 설정
ex) int scores[] = {87, 92, 100, 65, 70, 10, 96, 77};

기본 초기화
: 초기화하는 값이 배열의 용적보다 작을 때, 배열이 선언되는 위치와 상관없이 배열의 나머지 값이 기본값으로 채워짐

배열 요소 접근

- out of range 오류
: 배열의 용적보다 큰 인덱스의 요소에 접근 시 발생
→ 첨자 표현식에서 인덱스는 반드시 0과 (용적 -1) 사이의 값이어야 하며, 범위 이탈 시 쓰레기 값 추출 또는 오류가 발생한다.

변경 연산
→ 배열 요소의 값 또는 순서를 변경하는 연산

: 두 요소를 교환
2. 배열 정렬

포인터와 메모리

참조
- 참조 변수는 일반적인 변수와 다르게 메모리 위에 새로운 영역을 잡지 않는다.
- 원본 변수가 있어야만 생성할 수 있다.

참조 자료형
- 복합 자료형으로, int의 참조(int&), double의 참조(double&), bool의 참조(bool&)의 형태로 구성된다.
- 변수를 참조 변수와 연결 시 변수의 자료형과 참조 변수의 자료형이 일치해야 한다.

참조 관계
- 참조 변수가 선언되고 어떤 변수와 연결되면, 참조 관계는 스코프를 벗어나서 변수가 파괴되기 전까지는 계속해서 유지된다. → 참조 관계는 한 번 정의되면 변경 불가
- C++에서는, 변수와 참조 변수가 연결된 관계를 constant relation이라고 표현한다.

참조 다중성
- 참조 변수 하나를 여러 변수에 연결할 수 없다.

---

값 추출
- 참조 관계가 만들어지면, 원본 변수와 참조 변수 중에 아무 것이나 사용해도 값에 접근하여 값을 추출할 수 있다.

---

값 변경
- 일반적으로 참조 관계가 만들어지면 데이터 변수 또는 참조 변수를 사용하여 원본값을 변경할 수 있다.
- 원본 변수에 const 한정자가 붙으면 값 변경이 불가하다.

---

참조로 전달
→ 참조를 활용해 함수의 커뮤니케이션이란 데이터를 전달하고 리턴하는 것

- 값 전달 특징
- 호출하는 함수 쪽 값이 호출되는 함수쪽의 변수로 복사된다.
- 값으로 전달에서는 인수와 매개변수가 독립적으로 존재한다.
- 매개변수를 변경하는 일이 인수를 변경하는 일에 영향을 주지 않는다.
- 복사 대상 객체가 크다면, 인수를 매개변수로 복사할 때 많은 복사 비용이 발생
- 참조 전달 특징
- 메모리를 추가로 할당하지 않는다.
- 인수와 매개변수가 동일한 객체이므로 따로 복사할 필요가 없어 복사 비용이 발생하지 않는다.
- 참조 전달 시 매개변수에 값을 지정할 수 없다.

---

- 참조 리턴 특징
- 참조 리턴은 복사 비용은 없으나 일반적인 매개변수와 지역 변수의 값 리턴 불가능

1. 변수 주소
- 변수는 1바이트 이상의 메모리를 가진다.
- 불과 문자는 1바이트를 차지하며, 정수와 부동 소수점은 4바이트 또는 그 이상을 차지한다.
- sizeof 연산자를 사용하면 몇 바이트를 차지하는지 알 수 있다.
- 변수 주소는 변수가 차지하는 첫 번째 바이트의 주소를 의미한다.
- 변수의 주소를 추출하려면 주소 연산자 (&)를 사용한다.

---

1. 주소
- 16진법으로 출력
- 포인터 자료형과 포인터 변수를 활용하여 주소를 조작할 수 있다.

---

1. 포인터 자료형
- 리터럴의 값이 주소인 복합 자료형

---

1. 포인터 변수
- 포인터 변수에는 포인터 자료형의 값을 저장할 수 있다.
- 어떤 자료형의 포인터라도 포인터 자료형은 4바이트
- 선언: 포인터 변수 선언 시 자료형을 지정해야 함
- 초기화
: 리터럴 주소로는 초기화하지 않고, 기존 변수의 주소로 초기화

---

1. 간접 참조
- 포인터 변수를 선언하고 초기화했다면, 포인터 변수에 저장된 주소를 활용해서 포인트 된 변수의 값에 접근할 수 있다.
이 때 ‘*’ 기호를 사용하는 간접 참조 연산자를 사용한다.

연산자 정리

| 이름 | 연산자 | 표현식 | 우선 순위 | 결합 방향 |
| --- | --- | --- | --- | --- |
| 주소 | & | &value | 17 | ← |
| 간접 참조 | * | *pointer | 17 | ← |

---

& 기호는 참조 자료형을 정의할 때 사용한다.
변수의 주소를 얻기 위한 단항 연산자로 사용할 수도 있다.

포인터 자료형을 정의할 때는 자료형 뒤에 int* 형태로 배치한다.

간접 참조할 때는 포인터 앞에 붙인다.(*pScore)

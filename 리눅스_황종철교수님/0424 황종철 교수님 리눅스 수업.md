프로세스

- 각 프로세스는 고유한 번호를 가진다(PID)
- 프로세스 종류: 데몬, 고아, 좀비
- 데몬 프로세스: 특정 서비스를 제공하기 위해 존재하며 리눅스 커널에 의해 실행
- 고아 프로세스: 자식 프로세스보다 부모 프로세스가 먼저 종료되면 발생
- 좀비 프로세스: 자식이 실행종료했는데도 테이블 목록에 남아 있는 경우

프로세스 목록 확인 명령어: ps

- 일반적으로 ps -ef를 가장 많이 씀.

특정 프로세스 정보 검색: pgrep

- pgrep -x bash

---

시그널

- 프로세스에 무언가 발생했음을 알리는 간단한 메시지
- 무엇이 발생했는지를 나타내는 미리 정의된 상수 사용
- 시그널은 번호로 구분되며 이름을 가지고 있음
- 시그널을 받은 프로세스는 기본적으로 종료
- 주요 시그널: SIGHUP, SIGINT, SIGQUIT, SIGKILL, SIGALRM, SIGTERM

+ 명령을 백그라운드로 실행하려면 명령의 마지막에 & 기호를 추가

작업 목록 확인 명령어: jobs [%작업 번호]

ext4 파일 시스템 구조
—> 저장 장치를 논리적인 블록의 집합으로 구분
—> 일반적으로 블록은 4KB 크기이고 조정 가능
—> 블록 그룹 개수 = 장치 크기 / 블록 그룹의 크기

파일 시스템 마운트

- 마운트
→ 파일 시스템을 디렉터리 계층 구조의 특정 디렉터리와 연결
→ 연결되지 않으면 사용자가 해당 파일 시스템 접근 불가
- 마운트 포인트
→ 디렉터리 계층 구조에서 파일 시스템이 연결되는 디렉터리
- 파일 시스템 마운트 설정 파일 : /etc/fstab
→ 파일 시스템의 마운트 설정 정보 저장
→ 부팅 시 이 파일을 읽고 파일 시스템 마운트

```cpp
cat /etc/fstab

/dev/disk/by-uuid/31a9e662-6aa4-4952-a724-8b52e7f0063d / ext4 defaults 0 1

```

리눅스 시스템 부팅

과정
전원 - 바이오스 - 부트 로더 - 커널 초기화 - systemd 서비스 - 로그인 프롬프트 출력

1. 바이오스 단계
→ pc에 장착된 기본 하드웨어 상태 확인 후 부팅장치를 선택하여 부팅 디스크의 첫 섹터에서 512B를 로딩
→ 512B에는 디스크의 어느 파티션에 2차 부팅 프로그램이 있는지에 대한 정보가 저장되어 있음
→ 메모리에 로딩된 512B는 부트 로더를 찾아 메모리에 로딩작업 수행
2. 부트 로더
→ 여러 운영체제 중 부팅 운영체제 선택 메뉴 제공
3. 커널 초기화
→ 부트 로더에 의해 메모리에 로딩된 커널은 가장 먼저 시스템에 연결된 메모리, 디스크, 키보드, 마우스 등 장치 검사
→ 장치 검사 등 기본적인 초기화 과정이 끝나면 커널은 프로세스와 스레드 생성
4. systemd 서비스
→ 리눅스 동작

systemd 서비스

역할
→ 리눅스의 시스템과 서비스 관리자
→ 유닉스의 init 프로세스가 하던 작업 대신 수행
→ 다양한 서비스 데몬 시작 및 프로세스 상태 유지, 시스템 상태 관리

장점
→ 소켓 기반으로 동작, inetd 와 호환성 유지
→ 셸과 독립적 부팅 가능
→ 마운트 제어 가능
→ fsck 제어 가능
→ 시스템 상태에 대한 스냅숏 유지
→ 서비스에 시그널 전달 가능
→ 셧다운 전에 사용자 세션의 안전한 종료 가능

systemd 유닛
→ systemd는 유닛이라 부르는 구성 요소를 사용
→ 관리 대상의 이름을 ‘서비스명.유닛 종류’ 의 형태로 관리

systemd 관련 명령
→ systemctl : systemd 서비스 제어
-a : 유닛 전체 출력
-t <유닛 종류> : 지정 유닛만 출력

리눅스 종료
→ shutdown ( sudo shutdown -h now )
+ sudo shutdown -r : 시스템 재시작
+ sudo shutdown -c : 명령 취소
+ sudo shutdown -k : 메시지만 보내기
→ halt ( ls -l /sbin/halt )
→ poweroff ( ls -l /sbin/poweroff )
→ reboot ( ls -l /sbin/reboot )

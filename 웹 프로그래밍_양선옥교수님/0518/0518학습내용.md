- Node.js

1. 모든 웹 브라우저에서 작동
2. 실행 결과 즉시 확인 가능

입력과 출력

- 콘솔 창에서 자바스크립트 사용
1) 웹 브라우저 주소 표시줄에 about:blank 입력
2) 웹 개발자 도구 창에서 [콘솔] 선택 ( 컨트롤 쉬프트 제이)
- alert() 함수
: 알림 창 표시, 괄호 안에 메시지 입력 및 변수 사용
- confirm() 함수
: 확인 창 표시, 확인/취소 버튼이 있어서 사용자가 어떤 버튼을 클릭했는가에 따라 다르게 동작시킬 수 있음
- prompt() 함수
: 프롬프트 창은 사용자가 간단한 값을 입력할 수 있는 창 표시
프로그램 실행에 필요한 값을 받을 때 사용
- console.log() 함수
: 콘솔 창에 괄호 안 내용 표시
- document.write() 함수
: 괄호 안의 내용을 웹 브라우저 화면에 표시

웹 문서에서 스크립트 작성

- 자바스크립트 처리기는 웹 브라우저 안에 포함되어 있기 때문에, 자바스크립트 소스는 웹 문서에 작성하면 웹 브라우저가 해석함

스크립트 작성 위치

1. 인라인 스크립트
- HTML 태그 안에 직접 작성하는 자바스크립트

```html
<body>
	<button onclick = "alert("알림 메시지가 표시됩니다.")">클릭!</button>
</body>
```

1. 내부 스크립트
- 웹 문서에서 <script> 태그와 </script> 태그 사이에 실행할 자바스크립트 소스 작성
- 내부 스크립트는 주로 </body> 태그 앞에 사용

인라인과 내부 스크립트의 단점
: 같은 자바스크립트 소스를 여러 웹 문서에서 사용해야 할 경우, 필요한 문서마다 똑같은 소스를 반복해서 삽입해야 한다.

자료형

특수 기호 표시
: 특수 기호 표시할 때, 백슬래시(\) 다음에 기호 사용
ex) \ddd —> 8진수 문자
      \xddd —> 16진수 문자

템플릿 리터럴
: 문자열과 변수, 식을 섞어서 하나의 문자열을 만드는 방식
  백틱(` `) 기호를 사용해 문자열을 만든다.
  템플릿 리터럴 안의 변수나 식은 ${ } 로 묶어서 표현한다.

- truthy & falsy
: truthy는 참으로 인정할 수 있는 값, falsy는 반대
- undefined ( 사용자가 의도하지 않은 것)
: 변수를 선언하기만 하고 값을 할당하지 않을 때 변수 초깃값
  값이면서 동시에 자료형이다.
- null
: 유효하지 않은 값 ( 사용자가 의도한 것 )
  값이면서 동시에 자료형이다.

자료형 변환

- 느슨한 자료형 체크?
: 자바스크립트는 미리 변수의 자료형을 지정하지 않음
  변수에 값을 할당할 때 자료형이 결정됨
- 자동 형 변환
: 컴퓨터가 계산에 따라 자동으로 형 변환을 시킴
- 자료형 변환 함수
1) Number(): 문자열이나 논리형 값을 숫자로 변환
2) parseInt(): 문자열을 정수 숫자로 변환
3) parseFloat(): 문자열을 실수 숫자로 변환
4) String(): 숫자나 논리형 값을 문자열로 변환
5) Boolean(): 괄호 안의 값을 논리형으로 변환

조건문

if 문, if else 문

```jsx
// 3의 배수 확인
<script>
	let userNumber = parseInt(prompt('숫자를 입력하세요. '));
	if (userNumber % 3 === 0)
		alert('3의 배수입니다.');
	else
		alert('3의 배수가 아닙니다.');
</script>
```

중첩 if ~ else 문

```jsx
 /* 사용자가 입력한 숫자가 3의 배수인지 판별하기
    사용자가 '취소' 버튼을 눌렀는지(null) 확인
    '확인' 버튼 눌렀을 경우(null 이 아닐 경우) 
    -> 입력이 있으면 3의 배수 여부 판단 */
<script>
  let userNumber = prompt("숫자를 입력하세요. ");

  if (userNumber !== null) {
    if (parseInt(userNumber) % 3 === 0) alert("3의 배수입니다.");
    else alert("3의 배수가 아닙니다.");
  } else alert("입력이 취소되었습니다.");
</script>
```

변수의 유효 범위, 스코프

스코프 : 변수 적용 범위
* var 키워드를 사용한 변수는 함수 레벨 스코프
* let, const 키워드를 사용한 변수는 블록 레벨 스코프

var 예약어를 사용한 변수

```jsx
var sum = 0;  // 전역 변수 선언
function addNumber() {
  var result; // 지역 변수 선언
  sum = 10 + 20;
  result = 10 * 20;
}

addNumber();
console.log(sum);
console.log(result);
```

var 변수와 호이스팅?
호이스팅
→ 변수를 나중에 선언하지만, 내부적으로는 먼저 선언한 것처럼 인식하는 것
→ undefined 값으로 인해 예상치 못한 결과가 발생할 수 있다.
→ var 변수 사용 시 호이스팅이 발생하면 안된다.

const를 사용한 변수 특징

스마트 자판기 만들어보기

```jsx
const menu = {
        1: { name: "콜라", price: 1500 },
        2: { name: "사이다", price: 1300 },
        3: { name: "물", price: 800 },
        4: { name: "종료" },
      };
      console.log("자판기 메뉴");
      console.log(menu[1].name + ": " + menu[1].price + "원");
      console.log(menu[2].name + ": " + menu[2].price + "원");
      console.log(menu[3].name + ": " + menu[3].price + "원");
      console.log(menu[4].name);
      let money = prompt("자판기에 돈을 넣으세요: ");
      let balance = Number(money);

      while (true) {
        let promptMessage = `[스마트 자판기]\n현재 잔액: ${balance}원\n\n
        구매하실 메뉴 번호를 입력하세요:\n
        1. 콜라 (1500원)\n2. 사이다 (1300원)\n3. 물 (800원)\n4. 종료`;
        let choice = prompt(promptMessage);

        if (choice === "4") {
          console.log("자판기 프로그램을 종료합니다");
          break;
        }
        if (menu[choice]) {
          let selectedItem = menu[choice];

          if (balance >= selectedItem.price) {
            balance -= selectedItem.price;
            alert(`[구매 완료] ${selectedItem.name}가 나옵니다.
                (남은 잔돈: ${balance}원`);
          } else {
            alert(`[잔액 부족] ${selectedItem.name}을(를) 결제할 수 없습니다.
                (현재 잔액: ${balance}원)`);
          }
        } else {
          alert("정해진 번호 만 입력해주세요.");
        }
      }
```

함수 표현식

익명 함수
: 함수 이름이 없는 함수
→ 익명함수는 마치 하나의 값처럼 사용하기 위해 사용한다.
→ 함수를 변수에 할당할 수도 있고, 다른 함수의 인자로 사용할 수도 있다.
→ 변수에 할당된 익명 함수는 변수를 이용해 함수를 실행

```jsx
let sum = function(a, b){
	return a + b;
}
document.write(`함수 실행 결과 : ${sum(10, 20)}`);
```

---

즉시 실행 함수
→ 선언과 동시에 실행
→ 식 형태로 선언하기 때문에 함수 선언 끝에 세미콜론을 붙인다.

```jsx
(function () {
  let userName = prompt(`이름`);
  alert(`${userName}님, 안녕`);
})();
```

화살표 함수
→ 익명 함수에서만 사용할 수 있다. ( 표기는 => )

```jsx
const hi = () => { return alert("안녕하세요")};
// 실행할 명령이 한 줄이면 중괄호 생략 가능
// return 문도 생략 가능
```

```jsx
// 매개변수가 있을 경우

```
